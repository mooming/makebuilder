# MakeBuilder TODO - Feature Suggestions

## High Priority

### 1. Parallel CMake Generation
- Generate multiple CMakeLists.txt files in parallel
- Use multi-threading to speed up large projects

### 2. Incremental Build Support
- Track which modules have changed since last run
- Only regenerate CMakeLists.txt for modified directories

### 3. Custom Build Type Templates
- Allow users to define custom build type templates
- Support for custom compiler flags per build type

### 4. Multi-Configuration Build Support
- Generate executables for debug, dev, and release configurations
- debug: debug symbols, no optimization
- dev: optimized with debug information (-O1 or -O2 with -g)
- release: full optimization, no debug symbols
- Use CMake multi-config generator support (--config flag)

## Medium Priority

### 4. Visual Project Viewer
- Generate an HTML/JSON summary of project structure
- Show module dependencies, include paths, and build types

### 5. IDE Integration
- Generate `.vscode` or `.idea` workspace files
- Support for VS Code, CLion, Qt Creator

### 6. Dependency Graph Output
- Generate DOT/PlantUML dependency graphs
- Help visualize project structure

### 7. Header-Only Library Auto-Detection
- Automatically detect header-only libraries based on content
- No .module.config needed if only headers present

### 8. Custom Output Directory Support
- Support custom build directories per module
- Support for out-of-source builds

## Low Priority

### 9. Watch Mode
- Monitor file system for changes
- Auto-regenerate CMakeLists.txt on file changes

### 10. Script Pre/Post-Generation Hooks
- Run custom scripts before/after CMake generation
- Support for code generation tools

### 11. Multi-Language Support
- Support for C, Objective-C, CUDA
- Support for mixed-language projects

### 12. Test Discovery
- Automatically detect and configure tests
- Support for Google Test, Catch2, CTest

### 13. Package Manager Integration
- Support for FetchContent_Declare
- Support for vcpkg, Conan integration

### 14. Verbose Mode / Dry Run
- Show what would be generated without writing files
- Help with debugging configuration issues

### 15. Configuration Validation
- Validate .project.config and .module.config schemas
- Show helpful error messages for common mistakes
